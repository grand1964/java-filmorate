## *Пояснения к схеме.*

1. Дружба является отношением "многие ко многим". Для его реализации введена таблица friends, оба поля которой связаны по внешнему ключу с полем id таблицы users.
2. Лайки - отношение "один ко многим". Для них создана таблица likes, а ее поле film_id связано по внешнему ключу с полем id таблицы films. Связь таблиц likes и users не предусмотрена, поскольку по условию задания пользователи, ставящие лайки, не обязаны существовать.
3. Фильмы и жанры также связаны отношением "многие ко многим". Эта связь осуществляется через таблицу film_genres, функции которой аналогичны friends. Ее поля связаны по внешнему ключу с идентификаторами таблиц films и genres.
4. Поле id таблицы рейтингов mpa логически связано с полем mpa_id таблицы films. Внешний ключ здесь не установлен, поскольку формально фильм может не иметь рейтинга.
